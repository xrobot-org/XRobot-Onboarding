## 三种常见运行环境
### 裸机（Bare-metal）
- 没有操作系统，只有 `main()` 和一些中断函数。
- 程序一般是 `while(1)` 死循环，所有调度逻辑都要自己写。

### RTOS（实时操作系统，比如 FreeRTOS / ThreadX）
- 跑在 MCU 上的一个小型操作系统。
- 提供「任务/线程」「延时/定时」「队列/信号量」等基础能力。
- 好处：可以把不同功能拆成多个任务，而不是全塞在一个大循环里。

### Linux / 通用操作系统
- 常见在树莓派、工控机、x86 开发板等。
- 有进程/线程、文件系统、网络等完整功能。
- 我们的程序通常是用户态进程，通过系统调用来使用内核能力。

## 和 LibXR 的关系（先知道有这么一层就行）
- 这三种环境的底层 API 差别很大：
  - 裸机几乎没有线程，只剩下循环 + 中断。
  - RTOS 用 `xTaskCreate` / `tx_thread_create` 之类开任务。
  - Linux 用 `pthread_create` / `std::thread` 之类。
- LibXR 会在后面的「系统层」里，用统一的接口（`Thread` / `Mutex` / `Semaphore` / `Async` / `Timer` 等）把这些差异藏起来。
- 现在你只需要知道：以后会有这么一层帮你做「跨平台统一」，本任务不用看这些细节。
