## 本任务目标
- 选定后续示例要使用的 STM32 开发板（或自制板）。
- 选定并确认将要使用的调试器类型（建议 ST-LINK 或 J-LINK，DAPLink 也可以）。
- 完成开发板与调试器的物理连接，确认供电正常，为后续调试和下载固件做准备。
## 调试器类型概述
- ST-LINK：
  常见于 STM32 官方 Nucleo / Discovery 开发板上的板载调试器，也有独立版本。对 STM32 支持完善，是本路线优先推荐的选择。
- J-LINK：
  通用 SWD/JTAG 调试器，适用于多种 STM32 芯片，适合已经具备 J-LINK 或需要兼容多种芯片的场景。
- DAPLink：
  开源调试器方案，也可以用于 STM32 调试，但配置相对更麻烦。本路线在需要时会单独说明，这里只做类型确认和连接。
## 操作步骤
1. 确定开发板和芯片型号：
   - 在开发板丝印、芯片表面或板卡资料中找到完整的 STM32 型号，例如 STM32F103C8T6、STM32F407VG 等。
   - 记录下型号，后续生成工程和选择示例时会用到。
2. 确认调试器类型：
   - 如果是官方 Nucleo / Discovery 开发板，通常自带板载 ST-LINK，默认采用板载调试器即可。
   - 如果使用独立的 ST-LINK / J-LINK / DAPLink，请确认手边的具体型号，并准备好与开发板连接所需的线材（如杜邦线、排线等）。
3. 确认 SWD 接口引脚：
   - 在开发板上找到调试接口（常见为 SWD 接口），确认至少包含以下信号：SWDIO、SWCLK、GND、3V3（或 VCC）。
   - 对照调试器侧引脚标记，规划好一一对应的连接方式，确保地线和电源引脚不会接反。
4. 连接并上电：
   - 将调试器与开发板按照 SWD 引脚对应关系连接好。
   - 通过 USB 线为开发板供电：
     - 若是带板载 ST-LINK 的开发板，一根 USB 线同时负责供电和调试连接。
     - 若是独立调试器，确认开发板本身也已正确供电（USB、外接电源等）。
   - 观察开发板和调试器上的电源指示灯是否点亮，确认硬件已经正常上电。
5. 记录当前使用的组合：
   - 记下“开发板型号 + 调试器类型”的组合，例如：STM32F103C8T6 + ST-LINK，或 STM32F407VG + J-LINK。
   - 后续所有示例建议尽量使用同一套组合，避免频繁更换带来的干扰。
## 注意事项
- 接线时务必确认 GND 和电源引脚，不要把信号线接到电源或相互短接。
- 如果使用 DAPLink，后续在软件侧通常需要配合 OpenOCD 等工具，本任务阶段只需完成硬件连接和供电确认，具体调试配置将放在后续步骤或专门章节中说明。
## 完成标准
- 已明确将要使用的 STM32 开发板与芯片型号。
- 已明确将要使用的调试器类型（ST-LINK / J-LINK / DAPLink 等）。
- 开发板与调试器连接完毕，接入电脑后电源指示灯正常亮起，为后续调试与下载固件做好硬件准备。
